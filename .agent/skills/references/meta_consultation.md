# Meta-skill consultation (shared)

When building plans, context packages, or task sequences:

1. **memory_manager** – Completed tasks, roadmap, outcomes. Check first to avoid duplicate work and align with priorities.
2. **dependency_analyzer** – Affected files, dependency chains, conflict risk. Use so steps don’t break dependencies.
3. **version_control_tracker** – Change history, rollback points. Use for risky or broad changes.
4. **testing_coordinator** – Scheduled/required tests. Use for medium/high-risk work so tests are included.

Consult only the ones relevant to the current task; include their summaries in notes/plan output when it helps execution skills.
